## Git 推送/克隆 GitHub 超时问题排查与修复

**症状**：`git push`/访问 `https://github.com/my-xr/*.git` 报 `Failed to connect to github.com port 443 after 21080 ms`，但浏览器开梯子能正常打开 GitHub。

**根因**：
1. Git 命令行默认不读系统代理变量，https 协议直连 `github.com:443` 被墙 → 超时。
2. GitHub 已不支持密码登录；且 Git Credential Manager 在非交互终端（IDE 内置终端）会段错误崩溃，弹不出登录窗口。

**最终解决方案（改用 SSH 协议，一劳永逸）**：
- 生成 ed25519 key：`ssh-keygen -t ed25519 -C "2752351109@qq.com" -f ~/.ssh/id_ed25519 -N ""`
- 在 `~/.ssh/config` 加 GitHub 走 443 隧道配置（绕开 22 端口封锁）：
  ```
  Host github.com
    HostName ssh.github.com
    Port 443
    User git
    IdentityFile ~/.ssh/id_ed25519
    IdentitiesOnly yes
  ```
- 公钥加到 GitHub 账户 → Settings → SSH and GPG keys
- remote 改 SSH：`git remote set-url origin git@github.com:my-xr/<repo>.git`
- 验证：`ssh -T git@github.com` 出现 `Hi my-xr!` 即成功

**已处理仓库**：D:\dev\resume、D:\dev\anda-demo 均已切换为 SSH remote。
以后新仓库若 https 超时，直接 `git remote set-url origin git@github.com:...` 复用同一 key 即可。
