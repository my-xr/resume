字节跳动：算法驱动，一面淘汰率最高

技术驱动型公司，注重算法能力、工程实践与快速学习。一面是"生死关"。

⚡字节跳动 ByteDance
难度 ★★★★★重算法重项目3-4 轮
一面淘汰率约 50% 基础 40% + 手撕 35% + 项目 25% 要求源码级理解 研发岗校招 4000+

面试风格：字节面试官不满足于"知道"，要求"理解到源码级别"。一面主要考察基础功力和算法能力，二面更深（并发/JVM/系统设计），三面综合（架构思维 + 场景设计）。几乎每场必出 1-2 道手撕算法，且要求写出可运行代码并分析复杂度。

① 高频八股

  • 并发：synchronized vs ReentrantLock 底层实现；volatile 作用与内存语义；ThreadLocal 原理与内存泄漏；线程池参数与拒绝策略；CAS/AQS
  • JVM：内存结构（哪些线程共享）；GC Roots；CMS vs G1 vs ZGC；Full GC 频繁排查
  • MySQL：事务隔离级别；B+ 树 vs B 树；MVCC
  • Redis：缓存三问；分布式锁（红锁）；持久化
  • MQ：Kafka 顺序性与可靠性；RabbitMQ 死信
  • 分布式：分布式 ID（雪花算法）；CAP；分布式事务

② 高频手撕算法

  • 手写 LRU 缓存（双向链表 + HashMap，O(1)）★
  • 反转链表 / 链表环检测 / 返回环入口节点 ★
  • 滑动窗口最大值（要求低复杂度）★
  • 前 K 个高频元素
  • 岛屿数量（DFS/BFS）
  • 完全平方数（BFS/DP）
  • 课程表（拓扑排序）
  • 无重复字符的最长子串

③ 系统设计 / 场景题

  • 设计评论系统（高并发写入 + 分页 + 热评 + 防刷）
  • 设计秒杀系统
  • 分布式 TopK 问题
  • 价格计算服务设计
  • 线上 CPU 100% / 内存泄漏排查

④ 项目深挖方向

  • 项目整体架构图（要能画）
  • 最难 bug / 性能问题（量化）
  • Redis 为什么不用本地缓存
  • 分布式锁/限流/幂等怎么实现
  • 多线程/线程池业务场景
⚠️ 挂人雷区
项目太平淡（无技术挑战）、算法卡壳写不出、八股只背结论不懂原理被追问翻车。
✓ 备考核心
LeetCode 热题刷 200+，每题能讲思路+复杂度；八股要能讲到源码级（如 HashMap 追问到底层数组+红黑树阈值+扩容过程）。
3

阿里巴巴：深度广度并重，系统设计是分水岭

技术深度和广度著称，注重基础深入理解 + 大规模分布式系统设计。AI 岗占比超 60%。

🧡阿里巴巴 Alibaba
难度 ★★★★☆重系统设计重项目3 轮技术
一面 项目深挖+基础(60min) 二面 系统设计+场景(45min) 三面 架构思维+深度(30min) 新增 Java21 虚拟线程/云原生

面试风格：阿里最看重"技术选型的合理性"和"性能优化方案"。一道题会从原理问到工程权衡（如缓存一致性从概念问到多级缓存、从 B+ 树问到优化器、锁机制、隔离级别）。系统设计题是大头，要求画出架构图并讲清权衡。

① 高频八股

  • JVM：内存区域与分代；GC 算法；CMS vs G1；Full GC 调优；OOM 类型；内存泄漏排查
  • 并发：synchronized 锁升级（偏向→轻量→重量）；悲观锁 vs 乐观锁；线程池设计（CPU/IO 密集型）；CountDownLatch/CyclicBarrier/Semaphore
  • 集合：HashMap 1.7 vs 1.8；ConcurrentHashMap；CopyOnWriteArrayList
  • 分布式：缓存一致性；分布式事务（TCC/Seata/最大努力通知）；CAP；消息可靠投递（ACK/事务消息）
  • 阿里技术体系：Dubbo、RocketMQ、Sentinel、Nacos

② 系统设计高频题

  • 千万级 QPS 短链系统设计 ★
  • 秒杀系统（限流/超卖/幂等/扣库存）★
  • 分布式锁优化方案对比
  • 多活架构设计难点
  • 混沌工程实施经验
  • 分布式 ID 生成器
  • 高并发下缓存一致性处理

③ 高频手撕

  • LRU 缓存
  • 单例模式（双重检测锁）
  • 常见 DP（爬楼梯/背包）
  • 链表操作
  • 算法难度中等，但更看重场景题

④ 项目深挖方向

  • 架构图与技术选型合理性
  • 高并发如何处理
  • 分布式数据一致性
  • 微服务调用失败如何处理
  • 性能瓶颈与量化优化结果
⚠️ 挂人雷区
不会画架构图、表达不清晰、项目讲不出量化指标（"提升了性能"而非"QPS 800→3200"）。
✓ 备考核心
每个项目准备"架构图 + 3 个技术决策 + 量化结果"；系统设计专项训练（短链/秒杀/限流）；分布式事务方案对比要能脱口而出。
4

百度：基础扎实，AI 方向占比极高

搜索引擎 + AI 公司基因，技术广度深度并重。未来 3 年 2.1 万实习岗，87% 与 AI 相关。

🔵百度 Baidu
难度 ★★★★重基础AI 占比高实习转正率高
87% 岗位与 AI 相关 大模型/机器学习/无人驾驶 实习转正率高，大三尽早投

面试风格：百度是典型的"基础派"，计算机四大件（数据结构/操作系统/网络/数据库）问得细。AI 相关部门会考察机器学习基础、大模型应用、向量检索等。对 C++/C 基础也有一定要求（搜索底层）。

① 高频八股

  • 基础：数据结构与算法复杂度；OS 进程/线程/内存；网络 TCP/IP/HTTP/HTTPS
  • Java：JVM、并发、集合（同通用后端）
  • 数据库：MySQL 索引/事务/锁；SQL 优化
  • 搜索相关：倒排索引原理；ES；分词
  • AI 方向：ML 基础（LR/SVM/树模型）；DL（CNN/RNN/Transformer）；LLM 应用；RAG；向量数据库

② 高频手撕 / 场景

  • LeetCode 中等题为主
  • 字符串处理、树遍历、图算法
  • SQL 手写（join/聚合/窗口函数）
  • 设计搜索引擎相关模块
  • 大模型推理加速（量化/蒸馏/LoRA）
⚠️ 挂人雷区
计算机基础不牢（尤其 OS/网络）；投 AI 岗但说不清 ML 基础概念。
✓ 备考核心
夯实四大件；投 AI 部门需补 ML/DL 基础 + 一个 RAG/Agent 项目；实习转正率高，大三尽早投暑期实习。

腾讯：细节控，轮次多、容错率低

综合型，基础面面俱到 + 手撕 + 软技能。流程长（最多 7 轮），但单题难度不算极高。

💚腾讯 Tencent
难度 ★★★★细节控7 轮拉锯容错率低
5 技术面 + GM + HR 985/211 + 大厂经历优先 3 年新增 2.8 万校招岗 毕业时间放宽至 2 年

面试风格：腾讯以"细节控"著称，八股考得很细，甚至会出现冷门考点（如 ThreadLocalMap 的开放寻址法实现）。轮次多所以面面俱到——编程题、前端/后端基础、项目、设计都涉及。难度不高但容错率低，问题没答到点上就挂。考心态。

① 高频八股（考得细）

  • OS：进程/线程；调度算法；死锁四条件与预防；虚拟内存；页面置换（LRU/FIFO）；Linux 内核（系统调用/中断）
  • 网络：TCP 三次四次（状态迁移/SYN 洪泛/TIME_WAIT 作用）；HTTPS（SSL/TLS 握手）；HTTP/2 多路复用
  • 冷门细节：ThreadLocalMap 开放寻址；C++ 内存模型；protected 作用域
  • Java：JVM/并发/集合（通用）
  • DB/Redis：MVCC、索引、缓存一致性

② 高频手撕

  • 字符串处理（高频）
  • 二叉树遍历
  • 图算法：Dijkstra 最短路径 / Floyd-Warshall
  • 回溯：N 皇后 / 全排列
  • DP：背包 / 最长公共子序列
  • 最长无重复子串

③ 项目 / 设计

  • 项目展开提问（leader 面）
  • 技术视野（总监面）
  • 职业规划 / 跳槽原因（GM 面）
  • 穿插编程题

④ 软技能

  • 英文自我介绍（高频卡壳点）
  • 表达流畅度
  • 职业规范
  • 抗压心态（轮次间隔随机）
⚠️ 挂人雷区
英文自我介绍卡壳、表达不流畅、问题答不到点上（容错率低）、心态崩（轮次间隔久）。
✓ 备考核心
多练英文自我介绍；八股要细（含冷门考点）；图算法/回溯/DP 专项；做好长期作战心态准备。

美团：场景题多，Redis/MySQL 是重头

缓存与数据库原理是核心，场景设计题占比高。校招 6000 人，技术岗持续扩招。

🟡美团 Meituan
难度 ★★★★☆重场景Redis/MySQL校招 6000
Redis 六大场景必背 Kafka 高频 场景设计题多 本地商业/到店/配送

面试风格：美团围绕 Redis、MySQL、Kafka 反复深挖，场景题非常多（秒杀、限流、订单超时）。二面会出现"全程项目 + 手撕"无八股的情况。手撕偏 DP 与限流类。

① 高频八股

  • Redis：集群模式与分片；RedLock；数据类型与底层；缓存三问；内存评估
  • MySQL：B+ 树；MVCC；索引失效；慢 SQL
  • Kafka：副本同步与 ISR；消息有序性；广播模式底层
  • 并发/JVM：volatile；CMS vs G1；G1 可预测停顿模型；CAS/ABA
  • 分布式：幂等性；分布式 ID；Service Mesh；CQRS；零信任；无锁队列
  • 网络：HTTP 1.1/2/3；TCP 连接复用；select/poll/epoll

② 高频手撕

  • 打家劫舍（DP）★
  • 最长公共子序列（DP）★
  • 反转链表（递归 + 迭代）
  • 手撕滑动窗口限流算法 ★
  • 滑动窗口线程安全
  • 两个有序数组原地合并
  • SQL 手写（分组取最新记录）

③ 场景设计题

  • 订单超时自动取消（RabbitMQ 死信）
  • 多商户订单分配公平性
  • 点赞防重复点击 / 并发
  • 热门帖子排名算法
  • 接口限流与熔断设计

④ 项目深挖

  • 为什么用 Redis，瓶颈在哪
  • Redis 前后请求时间对比
  • 为什么拆微服务
  • 查询接口 IO 时间与并发策略
⚠️ 挂人雷区
缓存/数据库原理说不清（Redis 集群、MVCC、索引失效）；场景题给不出方案。
✓ 备考核心
背熟 Redis 六大场景（三问/分布式锁/持久化/集群/淘汰/数据结构）；Kafka ISR/顺序性；手撕限流算法；准备 1-2 个带 Redis+MQ 的项目。

拼多多：八股深挖 + 算法 Hard，高薪高压

基础为王，八股问得细且深（追问到答不上来）。算法可达 Hard 级别。高薪但工作强度大。

🔴拼多多 PDD
难度 ★★★★★算法 Hard八股深高薪
笔试算法 Medium-Hard 追问到答不上来为止 JUC 源码要精读 会问加班/工作制度

面试风格：基础为王，八股（OS/网络/DB/数据结构）必问且很细很深——比如 MySQL 索引会从 B+ 树原理一路问到优化器选择、锁机制、隔离级别影响。几乎每轮技术面都含手撕，难度与笔试相当或略高，要求现场写出可运行代码并分析复杂度。项目深挖到技术选型/难点/优化点。

① 高频八股（深）

  • OS/并发：内存管理（页表/段页式）；coredump；多核进程线程；锁（自旋/互斥/CAS）；synchronized vs volatile；死锁四条件；JMM
  • 网络：TCP 三次四次；握手崩溃 vs 建立后崩溃；TIME_WAIT 优化；HTTPS 双重加密；TLS 握手与证书校验
  • MySQL：事务隔离行为差异；行锁 vs 表锁；分页查询优化；最左前缀；慢 SQL；explain 字段
  • Redis：数据结构场景；主从一致性；宕机恢复；集群哈希槽；缓存一致性；三问
  • MQ：RabbitMQ 死信；幂等；重复消费；顺序性；失败补偿

② 高频手撕（Hard 偏多）

  • 非递归二叉树中序遍历（栈模拟）★
  • 单链表检测环 / 链表解环（定位入口）★
  • 判断数组能否三等分
  • 数组去重排列（排列组合类）
  • 反转单链表
  • LRU 缓存
  • 括号匹配验证（栈）
  • 用队列实现栈 / 栈实现队列
  • 线程安全的单例模式 ★

③ 项目拷打方向

  • Redis 具体使用方式与选型依据
  • Redis 宕机数据恢复与丢失风险
  • SQL 优化实践与索引效果验证
  • 大 offset 分页查询优化
  • MQ 消息丢失/重复消费解决
  • CPU 100% 线上排查流程

④ 非技术问题

  • 用几个关键词描述自己
  • 如何看待加班文化
  • 是否了解拼多多工作制度
  • 对团队/业务的提问
⚠️ 挂人雷区
并发细节说不清（JUC 源码级）、算法 Hard 卡壳、八股只背概念不懂底层被追问翻车。
✓ 备考核心
精读 JUC 源码（AQS/ReentrantLock/线程池）；OS/网络底层（页表/TIME_WAIT/TLS）要细；算法刷到 Hard；项目能扛 30 分钟拷打。

小红书：分布式场景 + 非常规手撕，深度惊人
